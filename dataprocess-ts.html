"Trend (seasonality) extraction" [MEDIUM]¶

Candidate : Context and goal

Common finance applications imply the use of time series data. However, the widely known theory around ARMA models is based on the assumption that the described process is stationnary. Hence, the mandatory trend and seasonality extraction task lies with the analyst.

Candidate : Asked solution and method of evaluation

In this exercise, you will be asked to propose an algorithm to extract a linear trend (yearly seasonality) from a given Pandas.Series object, describing the daily opening value for the Google share, from July to November 2012. Your algorithm will return the untrended (deseasonalized) Series, and we will test the result for remaining trend (seasonality).

import numpy as np

def untrend_sol(s):
    
    s_modif = s.reset_index()
    
    # Extract coordinates
    x = s_modif.index.values
    y = s_modif["Open"]
    
    # Extract slope(a) and intercept(b) from linear regression
    # We use np.linalg.lstsq, following the example
    # M is [[x 1]] in the matrix repr of linear reg
    
    M = np.vstack([x, np.ones(len(x))]).T
    
    a, b = np.linalg.lstsq(M, y)[0]
    
    print("Slope : {0}\nIntercept : {1}".format(a,b))
    
    trend = a*x + b
    s_trend = pd.Series(trend, s.index)
    
    s_notrend = s - s_trend
    
    return s_notrend

def untrend_check(s):
    
    s_modif = s.reset_index()
    
    # Extract coordinates
    x = s_modif.index.values
    y = s_modif[0]
    
    # Extract slope(a) and intercept(b) from linear regression
    # We use np.linalg.lstsq, following the example
    # M is [[x 1]] in the matrix repr of linear reg
    
    M = np.vstack([x, np.ones(len(x))]).T
    
    a, b = np.linalg.lstsq(M, y)[0]
    
    print("Slope : {0}\nIntercept : {1}".format(a,b))
    
    return not( abs(a) > 1e-8 or abs(b) > 1e-8 )

"Velocity and acceleration in a curve" [EASY]¶

Candidate : Context and goal

Numerous financial models make use of physical principles. For example, we hear about "mean-reversion", "momentum" and more. In specific applications (such as Kinetic Component Analysis), we might consider that a given time series is a curve, and analyze with kinetic concepts : velocity and acceleration.

Candidate : Asked solution and method of evaluation

In this exercise, we will ask you to compute pointwise velocity and acceleration, assuming each point represents position. You should use simple numerical approximations, such as $v(t) = {p(t+h) - p(t) \over h} .$ We evaluate your output by comparing it to ours (we use the left-side approximation, i.e. $\dot{x}(t) = x(t) - x(t-1) $ ).

Candidate : Needed ressources

None required (maybe Pandas.Series documentation).

Candidate : Problem

You will apply these methods to real-world data, i.e. the daily closing share value for Google, from 2010 to 2013. These data are stored in a Pandas.Series object, and were extracted with the following :

In [23]:
# s_pos, s_vel, and s_acc are respectively the position,
# velocity and acceleration (as Pandas.Series).

# Velocity computation
def get_velocity_sol(s_pos):
    
    s_vel = s_pos.diff()
    
    return s_vel

# Acceleration computation
def get_acceleration_sol(s_pos):
    
    s_acc = get_velocity(
        get_velocity(s_pos)
    )
    
    return s_acc

def vel_acc_check(s_pos, s_vel, s_acc):
    s_vel_sol = get_velocity_sol(s_pos)
    s_acc_sol = get_acceleration_sol(s_pos)
    
    mse_vel = ((s_vel-s_vel_sol)**2).mean()
    mse_acc = ((s_acc-s_acc_sol)**2).mean()
    
    return mse_vel + mse_acc
