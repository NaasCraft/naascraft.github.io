Sujet : Queue d'attente

Projet réalisé par C. Cony, P. Carles et G. Demonet
Enseignant supervisant le projet : C. Findling

Enoncé¶

Une queue d'attente a les caractéristiques suivantes : le temps entre deux arrivées (notons-le delta) suit une loi exponentielle de paramètre lambda_1, et le temps de service d'un client (notons-le service) suit une loi exponentielle de paramètre lambda_2. On souhaite estimer la probabilité que le nombre de clients dans la salle d'attente dépasse un certain seuil critique n entre le temps 0 et le temps t.

  1. Simuler la queue d'attente, et expliquer comment varie cette probabilité en fonction de lambda_1, lambda_2, n et t.
  2. Dans le cas où cette probabilité est très petite, on se propose d'appliquer la méthode de l'entropie croisée. Expliquer comment appliquer cette méthode dans ce cas précis, et l'appliquer pour un certain jeu de paramètres lambda_1, lambda_2, n et t.
  3. Reprendre cet exercice en remplaçant les lois exponentielles par des lois de Weibul.

Note : une difficulté de l'exercice est que le nombre de variables devant être simulées entre le temps 0 et le temps t est lui-même aléatoire, bien expliquer comment vous avez traité ce problème.

Approche du sujet et méthode¶

La file d'attente de la première partie du sujet est du type classique M/M/1 (décrit ici), on supposera également ici que les clients sont traités un à un suivant la règle du First In First Out.

Une représentation markovienne du processus est la suivante (en prenant pour lambda lambda_1 et pour mu lambda_2) :

source: Wikipedia

La loi de Little nous donne le nombre moyen de clients dans notre file en régime stationnaire, à condition qu'elle soit stable. Sachant que l'espérance d'une exponentielle est l'inverse de son paramètre, le nombre moyen de clients en régime stationnaire est alors dans notre cas lambda_1/lambda_2.

Mais l'objet du sujet n'est pas le régime stationnaire mais l'ensemble des régimes, incluant le régime transient. Il semble que ce régime ait été traité analytiquement dans "Transient behavior of the M/M/1 queue : starting at the origin", J. Abate and W. Whitt, 1986. Nous pourrons peut-être comparer nos résultats aux formules explicites que les auteurs ont obtenues.

Simulation de la file d'attente¶

Pour simuler la file d'attente, on choisira d'employer une méthode séquentielle, similaire au cas réel (cette méthode intuitive est appelée Simulation à évènements discrets). Ainsi, le processus suivant décrit cette méthode (Reprendre sans variables) :

  1. Définition des paramètres (n, t, lambda_1, lambda_2), en fixant i, delta, et service à 0 pour initialiser (on suppose que 0 est le temps d'arrivée du premier client). On fixe aussi notre variable de comptage ql (pour queue length) à 1.
  2. Tant que (i < t) :
    • Si (ql > n), on conserve cette information dans une variable globale booléenne over.
    • Si (service = 0), alors : (cas où à l'étape d'avant, un client a été servi et part)
      • Tirer une valeur de service. (sinon, on garde la précédente)
    • Si (delta = 0), alors : (cas où à l'étape d'avant, un client arrive)
      • Tirer une valeur de delta. (sinon, on garde la précédente)
    • Si (delta < service), alors : (un client arrive avant qu'un autre soit servi)
      • On enlève delta de service. (temps restant avant le prochain servi)
      • On ajoute delta à i.
      • On réinitialise delta à 0.
      • On incrémente ql de 1.
    • Sinon si (ql > 0) : (un client est servi avant qu'un autre n'arrive, à condition qu'il y en ait un)
      • On enlève service de delta. (temps restant avant la prochaine arrivée)
      • On ajoute service à i.
      • On réinitialise service à 0.
      • On décrémente ql de 1.
    • Sinon : (on attend juste un nouveau client)
      • On ajoute delta à i.
      • On réinitialise delta et ql à 0 et 1 resp.

Ce processus de simulation, en l'itérant, nous permet d'estimer la probabilité de over, autrement dit la probabilité que la file ait dépassé le seuil n. Pour tirer selon les lois exponentielles ou de Weibull, on utilisera le module numpy.random, et plus particulièrement les fonctions exponential et weibull.

# Définition de la fonction de simulation de file d'attente
# avec lois exponentielles

# On ajoute une trajectoire de la file dans le temps, 
# avec chaque point défini par (temps, taille de la file, action)

def queue_exp(n, t, lambda_1, lambda_2, traj=False, times=False):
    # Initialisation
    i, delta, service, ql = 0, 0, 0, 1
    over = False
    
    if traj: trajectory = [[0, 1, "IN"]] 
    # On ne connait pas la taille a priori, donc liste
    
    if times: times1, times2 = [], []
    
    # Boucle
    while (i < t):
        # Test de dépassement
        if (ql >= n): 
            over = True
        
        # Tirage du temps d'arrivée du prochain client
        if (delta == 0):
            delta = exponential(scale=1/lambda_1)
            if times: times1.append(delta)
        
        # Tirage du temps de service si besoin
        if (service == 0):
            service = exponential(scale=1/lambda_2)
            if times: times2.append(service)
                
        # Cas 1 : un client arrive avant
        if (delta < service):
            service = service - delta
            i = i + delta
            delta = 0
            ql += 1
            cas = "IN"
            
        else:
            # Cas 2 : il n'y a pas de client, on l'attend
            if (ql == 0): # donc on ne change pas service
                i = i + delta
                ql = 1
                cas = "IN"
            
            # Cas 3 : un client est servi avant
            else:
                delta = delta - service
                i = i + service
                service = 0
                ql -= 1
                cas = "OUT"
        
        # On met la trajectoire à jour
        if traj: trajectory += [[i, ql, cas]]
        
    if not times:
        if traj:
            return over, trajectory
        else:
            return over
    else:
        return times1, times2, over

La simulation d'une file d'attente étant maintenant possible, il s'agit de construire un estimateur de Monte Carlo de la probabilité de dépasser le seuil n. On note que l'on n'a pas besoin d'intégrer de burn-in.

In [5]:
def mcProb(n, t, lambda_1, lambda_2, n_samples):
    count = 0.
    for k in range(n_samples):
        if queue_exp(n, t, lambda_1, lambda_2): count+=1.
    
    return count/n_samples

def generateX(t, lambda_1, lambda_2, n_samples):
    result = []
    for k in range(n_samples):
        _, traj = queue_exp(1, t, lambda_1, lambda_2, traj=True)
        result += [traj]
    return result

Puis on construit l'estimateur CMC comme plus tôt à partir d'une liste de trajectoires données. On calcule aussi la variance empirique de notre estimateur.

def testTraj(x, n):
    test=False
    for i in range(len(x)):
        if x[i][1] >= n:
            test=True
    return test

def subTraj(x, t):
    k = 0
    result = []
    while(k < len(x)):
        if x[k][0] < t:
            result.append(x[k])
        else:
            break
        k += 1
    return result

def estCMC(x_list, n):
    count = 0.
    for k in range(len(x_list)):
        if testTraj(x_list[k], n): count+=1.
    return count/len(x_list)

def varCMC(x_list, est, n):
    var_sum = 0.
    for k in range(len(x_list)):
        if testTraj(x_list[k], n):
            var_sum += (1-est)**2
        else:
            var_sum += est**2
    return var_sum/len(x_list)

def generateMatrixEst(nmax, trange, l1range, l2range): 
    #les xrange sont au format [min, max, n_steps]
    l_t, t_step = trange[2], (trange[1] - trange[0])/(trange[2]-1)
    l_l1, l1_step = l1range[2], (l1range[1] - l1range[0])/(l1range[2]-1)
    l_l2, l2_step = l2range[2], (l2range[1] - l2range[0])/(l2range[2]-1)
    #on initialise un array numpy à 4 dimensions (n, t, l1, l2)
    matrixEst = np.zeros((nmax, l_t, l_l1, l_l2),
                        dtype=np.float32)
    matrixVar = np.zeros((nmax, l_t, l_l1, l_l2),
                        dtype=np.float32)
    t, l1, l2 = trange[0], l1range[0], l2range[0]
    
    total_begin_time = time()
    for i in range(l_l1): #indice lambda_1 : i
        
        for j in range(l_l2): #indice lambda_2 : j
            
            begin_time = time()
            
            #on a t, l1, et l2 fixes, on peut générer des trajectoires.
            x_base = generateX(trange[1], l1, l2, 1000)
            print("Generation for (t, l1, l2) = ({0}, {1}, {2})".format(
                    trange[1], l1, l2))
            print(" --- completed in {0} secs\n".format(
                    time()-begin_time))
            
            for k in range(l_t): #indice t : k
                x_list = [subTraj(x, t) for x in x_base]
                
                for n in range(nmax):
                    estimator = estCMC(x_list, n+1)
                    matrixEst[n, k, i, j] = estimator
                    matrixVar[n, k, i, j] = varCMC(x_list, estimator, n+1)
                    
                t += t_step
            
            duration = time() - begin_time
            print("Completed for set l1 = {0}, l2 = {1}.".format(l1, l2))
            print("Duration : {0} seconds\n".format(duration))
            t = trange[0]
            l2 += l2_step
        
        l2 = l2range[0]
        l1 += l1_step
    
    print("Total completion time : {0} seconds\n".format(
            time()-total_begin_time))
    return matrixEst, matrixVar

Méthode de l'entropie croisée¶

Lorsque le seuil à dépasser n devient grand, la probabilité à estimer est très faible, l'évènement est très rare. Il s'agit alors d'utiliser la méthode CE pour estimer correctement cette probabilité.

Une approche mathématique du problème a été réalisée et est lisible ici. On tentera pour une prochaine fois d'exploiter le module sympy qui permet l'affichage de LaTeX dans le notebook.

Pour mettre en place cette méthode d'estimation, il va nous falloir construire :

  • La recherche de (lambda_1, lambda_2) optimaux pour minimiser l'entropie croisée :
    • Fonction de passage à itérer pour déterminer le couple suivant
    • Critère d'arrêt
  • Le calcul de p(n,t) avec cette nouvelle distribution

def nextLambda(n, t, W, X, pastLambda, originLambda):
    length = len(X)
    num1 = np.zeros(length, dtype=np.float32) 
    denum1 = np.zeros(length, dtype=np.float32)
    num2 = np.zeros(length, dtype=np.float32)
    denum2 = np.zeros(length, dtype=np.float32)
    k = 0
    
    for xi in X:
        times1, times2, over = xi
        ma, md = len(times1), len(times2)
        lkhratio = W(xi, originLambda, pastLambda)
        
        num1[k] = over * lkhratio * ma
        denum1[k] = over * lkhratio * sum(times1)
        num2[k] = over * lkhratio * md
        denum2[k] = over * lkhratio * sum(times2)
        
        k += 1
    
    return (np.sum(num1)/np.sum(denum1), np.sum(num2)/np.sum(denum2))

def lkhRatio_exp(x, l1, l2):
    times1, times2, _ = x
    l1_1, l1_2 = l1
    l2_1, l2_2 = l2
    ma, md = len(times1), len(times2)
    sa, sd = sum(times1), sum(times2)
    
    const = (l1_1/l2_1)**ma * (l1_2/l2_2)**md
    toexp = (l2_1-l1_1)*sa + (l2_2 - l1_2)*sd
    
    return const*np.exp(toexp)
